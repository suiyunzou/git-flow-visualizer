Git 常用命令和冲突解决方案

常用命令

初始化和配置

git init

初始化一个新的 Git 仓库。

$ git init

git config

配置 Git 设置。

$ git config --global user.name "Your Name" $ git config --global user.email "your@email.com"

基本操作

git add

将文件添加到暂存区。

$ git add filename $ git add .

git commit

提交暂存的更改。

$ git commit -m "Commit message"

git status

查看仓库状态。

$ git status

分支操作

git branch

列出、创建或删除分支。

$ git branch $ git branch new-branch $ git branch -d branch-name

git checkout

切换分支或恢复工作树文件。

$ git checkout branch-name $ git checkout -b new-branch

git merge

合并一个或多个分支到当前分支。

$ git merge branch-name

远程仓库操作

git remote

管理远程仓库。

$ git remote add origin https://github.com/user/repo.git $ git remote -v

git push

将本地分支的更新推送到远程仓库。

$ git push origin main

git pull

从远程仓库获取并合并更改。

$ git pull origin main

冲突解决

合并冲突

识别冲突

当 Git 无法自动合并分支时，会出现合并冲突。冲突的文件会包含特殊的标记。

<<<<<<< HEAD Your changes ======= Changes from other branch >>>>>>> branch-name

解决冲突

手动编辑冲突文件，选择要保留的更改。删除冲突标记，保存文件。

1. 打开冲突文件 2. 编辑内容，保留所需更改 3. 删除冲突标记 4. 保存文件

完成合并

解决所有冲突后，将更改添加到暂存区并提交。

$ git add . $ git commit -m "Resolved merge conflict"

Rebase 冲突

识别 Rebase 冲突

Rebase 过程中可能会出现冲突。Git 会暂停 rebase 操作，等待解决冲突。

$ git rebase main CONFLICT (content): Merge conflict in file.txt

解决 Rebase 冲突

解决冲突的步骤与合并冲突类似。解决后，继续 rebase 过程。

1. 编辑冲突文件 2. git add . 3. git rebase --continue

中止 Rebase

如果遇到问题，可以随时中止 rebase 操作。

$ git rebase --abort
